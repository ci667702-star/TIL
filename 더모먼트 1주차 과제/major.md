# 1. 각 전공 이해하기

## 프론트엔드(Front-end)
사용자가 서비스를 이용하기 위해 웹페이지나 앱 내에서 만나고 경험하는 모든 부분을 말한다. 즉, 사용자와 상호작용 하는 모든 부분을 말한다.

HTML, CSS, JavaScript 등의 언어를 활용하여 개발한다. 그 외에도 Bootstrap Foundation과 같은 프레임워크를 활용하거나 jQuery, React.js와 같은 라이브러리를 통해 효율적인 코드 작성을 하기도 한다.

**하는 일**
1. 사이트/앱 디자인 구현
    - 사이트 디자인과 디자인 구현은 다르다. 프론트 엔드 개발자의 역할은 주어진 디자인을 웹사이트나 앱에 잘 구현하는 것이다.
2. 고객이 사용하는 기능 구현
    - 쇼핑 카트, 결제하기, 검색, 이미지 확대 등등 웹사이트/앱 상엔 다양한 기능이 필요하다. 프론트엔드 개발자의 제일 중요한 역할이 바로 이런 기능을 구현하는 일이다.
3. 사이트/앱 최적화
    - 사이트/앱의 기능만 구현된다고 해서 사용자가 만족하지는 않는다. 로딩 속도와 반응 속도가 느리면 사용자의 불만이 쌓이고 심한 경우 사용자들은 사이트/앱을 쓰지 않게 된다. 사이트/앱의 성능을 최적화하기 위해서는 프론트엔드 개발자도 큰 몫을 담당한다.
4. 사이트/앱 보안 관련 기능 구현
    - 보안은 소프트웨어 모든 영역에 있어 다루어지고, 프론트엔드 개발자 역시 보안 관련 기능을 염두에 두고 구현해야한다.
5. 사용자 접근성 관련 기능 구현
    - 접근성은 최대한의 사용자가 웹 사이트나 앱을 쉽게 이해하고 사용할 수 있는 정도를 뜻한다. 구현 단계에서 그 디자인을 구현하면서 기본적인 접근성 표준을 준수해야한다.

- `반응형 웹: 다양한 기기나 브라우저의 크기에 맞게 구성이나 크기를 변경해가며 반응하는 웹문서 또는 이를 사용하는 기법을 말한다.`
- `뷰포트: 현재 화면에 보여지고 있는 영역을 의미한다.`

## 백엔드(Back-end)
백엔드는 데이터베이스와 서버 로직을 관리한다. 데이터의 CRUD(Create, Read, Update, Delete) 작업을 담당하며, 비즈니스 로직을 처리한다. 즉, 백엔드 개발이란 사용자가 볼 수 없는 영역인 데이터베이스나 서버를 관리하는 것을 말한다. 예를 들어, 웹 사이트에서 로그인하면 백엔드는 이 정보를 확인하고 맞는지 여부를 결정하는 일을 한다.

**하는 일**
1. 서버 개발 및 관리
    - 백엔드 개발자는 애플리케이션을 만들고 유지보수하는데 필요한 컴퓨터 시스템을 선택하고 설정하며, 이를 관리하고 안정적으로 운영한다. 클라우드 서비스를 활용하여 데이터를 저장하며, 이 모든 것을 조합해서 사용자에게 원활한 애플리케이션을 제공한다.
2. 데이터베이스 설계 및 관리
    - 백엔드 개발자는 데이터베이스를 설계하고 관리한다. 데이터베이스 스키마를 작성하고, 쿼리를 작성하여 데이터를 저장, 검색 및 조작한다. 또한, 데이터베이스의 성능을 최적화하고, 백업 및 복구 전략을 구축한다.
3. API 개발
    - 백엔드 개발자는 다른 시스템이나 애플리케이션과 상호작용하기 위한 API(Application Programming Interface)를 개발한다. 이를 통해 클라이언트 애플리케이션과 데이터를 주고받을 수 있으며, 데이터의 입력, 출력, 수정, 삭제를 처리하는 역할을 수행한다.
4. 보안 구현
    - 백엔드 개발자는 데이터와 사용자 정보를 안전하게 보호하기 위한 보안 기능을 구현한다. 인증과 권한 부여, 데이터 암호화, 보안 취약점 해결 등을 처리하여 시스템의 보안성을 강화한다.
5. 성능 최적화
    - 백엔드 개발자는 시스템의 성능을 개선하기 위해 최적화 작업을 수행한다. 데이터베이스 쿼리의 튜닝, 캐싱 기법의 사용, 서버 리소스 관리 등을 통해 애플리케이션의 응답 속도와 처리 능력을 향상시킨다.

- `비즈니스 로직(Business Logic): 특정한 기능 수행을 위해 데이터를 처리하는 실직적인 코드`

데이터를 저장, 관리하고 사용자가 서비스 사용에 필요한 부분을 지원하는 역할이다. 구체적으로는 응용 프로그램을 만들고, 데이터 검색, 저장 및 통합, API 작성 등이 있다. PHP, Node.js, Java, Python 언어 등을 활용한다.

## 클라우드
### 클라우드의 의미
광대한 네트워크를 통하여 접근할 수 있는 가상화된 서버를 통해 서버에서 작동하는 프로그램과 데이터베이스를 제공하는 IT환경을 의미한다. 클라우드를 이용하면 필요한 컴퓨팅 자원을 인터넷으로 쉽게 이용할 수 있다.

**하는 일**<br>
1. 인프라 구축
    - 개발한 서비스의 연속성과 고가용성이 유지될 수 있도록 튼튼한 인프라 서비스 환경을 구축하고 운영한다.
    - 하이브리드/멀티 클라우드 인프라를 구축
2. 자동화
    - 서비스 개발 환경과 업무 생산성을 향상시키기 위해 자동화 기술활동을 통해 클라우드와 기업 비즈니스에 적합한 인프라 서비스 환경을 꾸준히 고도화 개선한다.
    - IaC(Infrastructure as Code) 기반으로 인프라를 코드로 관리하고, CI/CD 파이프라인을 구축해 배포 자동화를 진행한다.
3. 보안 및 비용 최적화
    - 클라우드 환경의 보안을 강화하고 IAM 설정, 네트워크 보안, 데이터 암호화를 적용하며, 비용 절감을 위한 모니터링과 최적화를 수행한다.
4. 클라우드 마이그레이션
    - 데이터, 애플리케이션, IT 리소스와 같은 디지털 자산을 클라우드로 이전하는 프로세스이다.

**기술**
- AWS
    - 클라우드 컴퓨팅을 통해 전 세계적으로 연결된 원격 컴퓨터를 활용하여 다양한 IT 리소스를 제공하는 주문형 서비스이다. 정교하고 확장성이 높은 애플리케이션을 구현할 수 있고, 자동으로 리소스를 관리해준다. 그리고 신뢰성과 가용성이 높다.
- Docker
    - 컨테이너 내에서 애플리케이션 배포 및 관리를 자동화할 수 있는 오픈 소스 플래폼이다. 어느 환경에서든 도커를 이용하면 서비스를 개발했을 때 환경과 똑같이 복제되어 실행이 용이하다.

## 디자인
웹/앱 서비스의 사용자 경험(UX)과 화면 인터페이스(UI)를 설계하고, 시각적으로 구현하여 편리하고 아름다운 디지털 환경을 만드는 전문가이다.

**UI(User Interface)**<br>
UI는 사람과 컴퓨터 시스템 사이의 커뮤니케이션을 의미한다. 그래서 우리가 웹사이트, 앱, 디스플레이 화면 등을 처음 접한 순간 보이는 모든 디자인 또는 기술적 요소를 UI라고 할 수 있다. UI는 크게 입력과 출력으로 사용된다. 입력은 사용자가 시스템을 조작하는 행위를, 출력은 그 사용자가 입력한 값에 따라 나오는 결과물을 말한다.

**UX(User Experience)**<br>
사용자 경험 이라는 뜻이 담긴 것 답게, 사용자의 모든 경험과 감정을 가리킨다. UX는 사용자가 경험한 모든 것을 가리키기 때문에 그 범위가 더 넓다. UX는 넓은 의미에서 UI도 포괄한다. 인터페이스가 직관적이고 깔끔하면 사용자는 편하게 웹/앱 서비스를 이용할 수 있고 좋은 사용자 경험으로 이어지기 때문이다.

**하는 일**
- UI 디자이너
    - GUI를 디자인
        - GUI란 사용자가 그래픽을 통하여 컴퓨터와 정보를 교환하는 인터페이스를 말한다.
        - 화면 내에서 사용자에게 정보를 보여줘야한다면 어떻게 보여줄지, 명령을 내렸을 때 네트워크 이슈 등으로 실행하지 못했다면 어떻게 알릴지 등 정보를 알려주는 방법을 디자인해야한다.
- UX 디자이너
    1. 사용성 개선
        - 얼마나 사용하기 편리한지 여부가 업무 목표이다.
    2. 사용자의 총체적 경험을 개선
        - 제품이나 서비스를 이용하며 오랜 시간 머물고 싶다는 감정이 생기고, 사용 빈도가 높아지며, 브랜드 자체 충성도도 높아지도록 하는 단계이다. 편리한 사용만을 목적으로 하는 1번과 비교하여 경험 자체를 깊게 다루는 방식이다.
    

## 협업
### 프론트엔드와 백엔드
프론트엔드와 백엔드는 API를 중간 다리로 삼아 서로 통신한다.

**API의 의미**<br>
API의 뜻은 응용 프로그램 인터페이스이다. 이 용어는 소프트웨어와 소프트웨어가 서로 데이터를 주고 받고, 기능을 공유할 수 있도록 돕는 규칙과 약속의 집합을 의미한다.

**작동 방식**
1. 클라이언트가 API를 통해 특정 요청을 보낸다. 이때 보통 URL 엔드포인트와 HTTP 메서드(GET, POST 등)를 사용한다.
2. 서버는 요청을 받아 필요한 데이터를 검색하거나 연산을 수행한다.
3. 서버는 처리 결과를 JSON 또는 XML 형식의 응답으로 클라이언트에게 반환한다.
4. 클라이언트는 응답 데이터를 활용해 사용자에게 결과를 보여주거나 추가 작업을 진행한다.

**역할**
역할 및 기능 | 설명 
:--|:--
시스템 연결 및 상호 운용성 | 서로 다른 소프트웨어 시스템을 연결하여 데이터 교환과 통합을 가능하게 함
데이터 공유 및 촉진 | 민감한 정보 접근을 통제하면서 협업과 데이터 공유를 지원
혁신 지원 | 외부 서비스와 데이터에 대한 접근을 제공하여 새로운 기능 개발 지원
사용자 경험 향상 | 다양한 서비스와 플랫폼을 연결하여 효율적이고 편리한 경험 제공

**종류**
1. RESTful API
    - 가장 일반적인 방법으로, 백엔드에서 RESTful 원칙을 따르는 API를 제공하고, 프론트엔드에서는 이 API를 호출한다.

    - `REST(REpresentational State Transfer): 통신을 통해 자원의 상태를 주고받는 것에 대한 가이드라인이다.`
2. GraphQL
    - 프론트엔드에서 필요한 데이터 형식을 지정하여 요청할 수 있다.
3. WebSocket
    - 실시간 통신이 필요한 경우에 사용되며 HTTP보다 더 빠른 양방향 통신을 지원한다.
4. Server-Sent Events(SSE)
    - 서버에서 클라이언트로 실시간으로 데이터를 전송할 때 사용한다.

구분 | REST | GraphQL
:--:|:--|:--
엔드포인트 | 여러 개의 엔드포인트 사용(/users, /orders 등) | 단일 엔드포인트 사용 (/graphql)
데이터 요청 | HTTP 메서드로 리소스 조작, 과다/부족 요청 문제 발생 가능 | 필요한 데이터만 요청, 네트워크 효율성 높음
설계 복잡성 | 설계와 구현이 쉬움 | 스키마 정의가 필요(어려움)
캐싱 지원| HTTP 표준 기반 캐싱 우수 | 캐싱 구현이 어려움
확장성 | 엔드포인트 증가 시 복잡성 증가 | 새로운 필드 추가 등 확장성 우수
실시간 지원 | 제한적 | Subscription을 사용하여 지원 가능
