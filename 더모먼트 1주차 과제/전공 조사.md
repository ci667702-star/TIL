# 1. 각 전공 이해하기

## 프론트엔드(Front-end)
사용자가 서비스를 이용하기 위해 웹페이지나 앱 내에서 만나고 경험하는 모든 부분을 말한다. 즉, 사용자와 상호작용 하는 모든 부분을 말한다.

HTML, CSS, JavaScript 등의 언어를 활용하여 개발한다. 그 외에도 Bootstrap Foundation과 같은 프레임워크를 활용하거나 jQuery, React.js와 같은 라이브러리를 통해 효율적인 코드 작성을 하기도 한다.

**하는 일**
1. 사이트/앱 디자인 구현
    - 사이트 디자인과 디자인 구현은 다르다. 프론트 엔드 개발자의 역할은 주어진 디자인을 웹사이트나 앱에 잘 구현하는 것이다.
2. 고객이 사용하는 기능 구현
    - 쇼핑 카드, 결제하기, 검색, 이미지 확대 등등 웹사이트/앱 상엔 다양한 기능이 필요하다. 프론트엔드 개발자의 제일 중요한 역할이 바로 이런 기능을 구현하는 일이다.
3. 사이트/앱 최적화
    - 사이트/앱의 기능만 구현된다고 해서 사용자가 만족하지는 않는다. 로딩 속도와 반응 속도가 느리면 사용자의 불만이 쌓이고 심한 경우 사용자들은 사이트/앱을 쓰지 않게 된다. 사이트/앱의 성능을 최적화하기 위해서는 프론트엔드 개발자도 큰 몫을 담당한다.
4. 사이트/앱 보안 관련 기능 구현
    - 보안은 소프트웨어 모든 영역에 있어 다루어지고, 프론트엔드 개발자 역시 보안 관련 기능을 염두에 두고 구현해야한다.
5. 사용자 접근성 관련 기능 구현
    - 접근성은 최대한의 사용자가 웹 사이트나 앱을 쉽게 이해하고 사용할 수 있는 정도를 뜻한다. 구현 단계에서 그 디자인을 구현하면서 기본적인 접근성 표준을 준수해야한다.

## 백엔드(Back-end)
백엔드는 데이터베이스와 서버 로직을 관리한다. 데이터의 CRUD(Creat, Read, Update, Delete) 작업을 담당하며, 비즈니스 로직을 처리한다.

**하는 일**
1. 서버 개발 및 관리
    - 백엔드 개발자는 애플리케이션을 만들고 유지보수하는데 필요한 컴퓨터 시스템을 선택하고 설정하며, 이를 관리하고 안정적으로 운영한다. 클라우드 서비스를 활용하여 데이터를 저장하며, 이 모든 것을 조합해서 사용자에게 원활한 애플리케이션을 제공한다.
2. 데이터베이스 관리
    - 백엔드 개발자는 데이터베이스를 설계하고 관리한다. 데이터베이스 스키마를 작성하고, 쿼리를 작성하여 데이터를 저장, 검색 및 조작한다. 또한, 데이터베이스의 성능을 최적화하고, 백업 및 복구 전략을 구축한다.
3. API 개발
    - 백엔드 개발자는 다른 시스템이나 애플리케이션과 상호작용하기 위한 API(Application Programming Interface)개발한다. 이를 통해 클라이언트 애플케이션과 데이터를 주고받을 수 있으며, 데이터의 입력, 출력, 수정, 삭제를 처리하는 역할을 수행한다.
4. 보안 구현
    - 백엔드 개발자는 데이터와 사용자 정보를 안전하게 보호하기 위한 보안 기능을 구현한다. 인증과 권한 부여, 데이터 암호화, 보안 취약점 해결 등을 처리하여 시스템의 보안성을 강화한다.
5. 성능 최적화
    - 백엔드 개발자는 시스템의 성능을 개선하기 위해 최적화 작업을 수행한다. 데이터베이스 쿼리의 튜닝, 캐싱 기법의 사용, 서버 리소스 관리 등을 통해 애플리케이션의 응답 속도와 처리 능력을 향상시킨다.

- `비즈니스 로직(Vusiness Logic): 특정한 기능 수행을 위해 데이터를 처리하는 실직적인 코드`

데이터를 저장, 관리하고 사용자가 서비스 사용에 필요한 부분을 지원하는 역할이다. 구체적으로는 응용 프로그램을 만들고, 데이터 검색, 저장 및 통합, API 작성 등이 있다. PHP, NOde.js, Java, Python 언어 등을 활용한다.
