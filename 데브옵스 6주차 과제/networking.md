# Networking

# DHCP(Dynamic Host Configuration Protocol)
DHCP는 호스트 IP 주소와 각종 TCP/IP 프로토콜의 기본 설정을 클라이언트에게 자동적으로 제공해주는 프로토콜을 말한다. DHCP는 네트워크 안에 컴퓨터에 자동으로 네임 서버 주소, IP 주소, 게이트웨이 주소를 할당해주는 것을 의미하고, 해당 클라이언트에게 일정 기간 임대를 하는 동적 주소 할당 프로토콜이다. 

**구성 요소**
- DHCP 서버
    - DHCP 서비스에서 실행되는 네트워크 장치이다. DHCP 서버는 IP 주소와 구성 관련 정보를 보유한다.
- DHCP 클라이언트
    - 네트워크에 연결하고 DHCP 서버와 통신하는 컴퓨터 또는 전화와 같은 장치이다.
- DHCP 릴레이
    - DHCP 클라이언트와 서버 간의 요청을 관리한다. 일반적으로 릴레이는 조직이 대규모 또는 복잡한 네트워크를 처리해야할 때 사용된다.

**동작 방식**
1. 공유기에 노트북을 LAN으로 연결하면 자동으로 DHCP Client의 MAC주소가 DHCP Server에게 IP 주소를 요청한다.
2. DHCP Server는 DHCP Client에게 사용 가능한 Ip를 자도으로 할당한다.
3. 단, DHCP를 통해 할당 받은 IP 주소는 영구적인 것이 아니다.
4. 할당 받은 IP 주소를 사용하다가 임대 시간이 지나면 기간을 연장하거나 IP 주소를 반납한다.


# **서브넷 마스크**
제한적인 자원을 낭비 없이 아껴쓰기 위한 방법 중 하나로 지금도 많이 사용되고 있다. 서브넷 마스크는 기본적으로 0과 1로 구성되어 있으며, IP 크기와 동일하게 32자리 2진수로 표현된다. 크기가 똑같은 이유는 IP주소와 서브넷 마스크를 AND 연산하기 위해서이다. 서븐세 마스크는 IP와 크기는 같으나 표현에 있어서 차이가 있다. 서브넷마스크는 1이 연속으로 나와야한다. 1101와 같은 표현은 없다. 무조건 0이 나오기 시작하면 그 뒤는 0이다.

IP 뒤에 /24라는 표현은 1이 24개가 있다는 뜻이다. 1은 네트워크 영역으로 사용하겠다는 뜻이고 0은 호스트 IP로 사용하겠다는 뜻이다. 즉 사용자에게 0이 표현된 부분만 호스트 IP를 할당할 수 있게 만들겠다는 뜻이다.

표기 | 서브넷 마스크 | 의미
:--:|:--|:--
/16 | 255.255.0.0 | 네트워크 16비트 + 호스트 16비트
/24 | 255.255.255.0 | 네트워크 24비트 + 호스트 8비트
/25 | 255.255.255.128 | /24를 반으로 나눔
/26 | 255.255.255.192 | /24를 4등분

구분 | 주소 범위 | 첫 번째 옥텟 | 용도

# CIDR 표기법

    192.168.10.70**/26**

뒤에 있는 /26이 CIDR이자 CIDR 표기법이다. 그리고 저런 식으로 표기하는 이유는 단 한줄만으로 네트워크 범위를 추측 또는 측정할 수 있기 때문이다.

기본적으로 CIDR은 `IP 주소/프리픽스 길이`로 표기한다.

클래스 | CIDR 표기
:--:|:--
클래스 A | 0.0.0.0/8 ~ 127.255.255.255/8
클래스 B | 128.0.0.0/16 ~ 191.255.255.255/16
클래스 C | 192.0.0.0/24 ~ 223.255.255.255/24
클래스 D | 224.0.0.0/4
클래스 E | 240.0.0.0/4



# 서브넷팅
서브넷팅이란 간단하게 말하자면 IP 주소를 효율적으로 나누어 사용하기 위한 방법을 일컫는다.

**서브넷팅의 목적**
1. 효율성
    - 관리가 편해지고 IP 주소를 효율적으로 나누어 낭비하는 것을 막을 수 있다.
    - 필요와 편의에 따라 구분하면 IP만을 통해 용도를 나눌 수도 있다.
2. 성능
    - 네트워크 브로드캐스트 사이즈를 줄일 수 있다.
    - 이는 조직 단위 사이즈를 줄여 브로드캐스트 내 트래픽을 줄여 성능을 향상시킨다.
3. 보안
    - 네트워크를 분리함에 따라 보안성이 향상된다.
    - 단일 도메인 내 여러 시스템이 존재하면 보안 이슈에 대한 파급 영향도 커진다.

**서브넷팅 문제**
- 인터넷 라우팅의 테이블이 증가하거나 인터넷으로부터 로컬 관리자의 네트워크 번호를 확인해야 한다.
- 서브넷을 나눌수록 각 네트워크에서 사용할 수 없는 주소가 늘어나 전체적으로 사용 가능한 IP 주소의 총량이 줄어든다.

# 서브넷팅 계산법

1. 서브넷 분할을 위한 최소 비트를 계산해야한다.
    - 2ⁿ ≥ 필요한 서브넷 수
2. 새로운 서브넷 마스크를 계산한다.
    - 기존 네트워크 마스크에 서브넷 비트를 추가하여 새로운 서브넷 마스크를 만든다.

\>
    원래 네트워크: 192.168.1.0/24

    11111111.11111111.11111111.00000000
    | 서브넷 비트 n개 추가
    V
    11111111.11111111.11111111.nnn00000

원래 마스크 + 서브넷 비트 => /24 + n

3. 각 서브넷의 호스트 수 계산
    - 호스트 비트 = 32 - 서브넷 마스크 길이 -2
    - 가장 첫번째 주소는 네트워크 주소, 마지막 주소는 브로드캐스트 주소로 예약된다.

# 문제

**문제 1** `192.168.1.0/24` 에서

- 네트워크 주소는? 
    - 192.168.1.0
- 브로드캐스트 주소는?
    - 192.168.1.255
- 사용 가능한 호스트 수는?
    - 254개이다.

**문제 2** `192.168.1.0/26` 에서

- 서브넷 마스크는?
    - 255.255.255.192
- 사용 가능한 호스트 수는?
    - 62개이다.
- 첫 번째 호스트 ~ 마지막 호스트 범위는?
    - 192.168.1.1 ~ 192.168.1.254

**문제 3** `172.16.0.0/16` 네트워크를 **4개의 서브넷**으로 나누려면

- 몇 비트를 빌려야 해?
    - 2비트를 빌린다.
- 각 서브넷의 네트워크 주소는?
    - 172.16.0.0
    - 172.16.64.0
    - 172.16.128.0
    - 172.16.192.0
- 각 서브넷당 호스트 수는?
    - 16382 개이다.
