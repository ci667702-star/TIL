# OSI 7계층 각 층에서 사용되는 기술 작성, 영어 포함
OSI 7계층은 네트워크 통신이 일어나는 과정을 7단계로 나눈 것을 말한다.

계층을 나눈 이유는 통신이 일어나는 과정을 단계별로 파악할 수 있기 때문이다. 흐름을 한눈에 알아보기 쉽고, 사람들이 이해하기 쉽고, 7단계 중 특정한 곳에 이상이 생기면 다른 단계의 장비 및 소프트웨어를 건드리지 않고도 이상이 생긴 단계만 고칠 수 있기 때문이다.

계층 | 계층명 | 기능 | 프로토콜
:--:|:--|:--|:--
7 | 응용 계층<br>(Application Layer)|사용자와 직접 상호작용하는 소프트웨어를 지원 사용자(응용 프로그램)을 OSI 환경에 접속 시킴 | HTTP, FTP, SMTP, DNS, Telnet, SSH
6 | 표현 계층<br>(Presentation Layer)|데이터 표현과 변환을 담당하여 데이터를 으용 계층으로 전달하고 응용 계층으로부터 받은 Data를 수신측에 맞는 코드/형식으로 변환. 혹은 반대의 과정을 수행한다. 필요시 암호화(또는 복호화)를 진행 | JPEG, MPEG, SSL, TLS
5 | 세션 계층<br>(Session Layer)| 통신을 위한 세션 관리, 연결 설정 및 해체, 동기화<br>송신시: 데이터 복구를 위한 동기점(대동기점, 소동기점) 생성<br>수신시: 동기점 확인|NetBIOS, RPC, NFS
4 | 전송 계층<br>(Transport Layer)|포트번호를 관리하여 수신된 데이터가 어느 응용 프로그램에 전송될지 판독, 데이터 전송<br>전송사용 프로토콜 결정(TCP or UDP)<br>신뢰성 있는 통신 서비스를 제공하며, 패킷 분할 및 재조립, 오류 검출 및 복구 | TCP, UDP
3 | 네트워크 계층<br>(Network Layer)|패킷을 목적지까지 전달하고, 경로 선택 및 데이터 전송을 위한 논리적 주소 지정<br>네트워크에서 목적지로 데이터 전송을 위해 IP 주소를 이용해 최적의 경로를 찾고(routing), 다음 라우터에 데이터를 넘겨줌(forwarding) | IP, ICMP, ARP, RARP
2 | 데이터 링크 계층<br>(Data Link Layer)|LAN 내부 같은 네트워크에 있는 여러대 컴퓨터 데이터 교환 프레이밍이 일어남<br>전송단위:프레임<br>물리계층으로부터 전송된 데이터의 오류를 검출하고 수정하며, 데이터 전송을 위한 물리적 주소(MAC) 지정 | Ethernet, Token Ring, FDDI, HDLC
1 | 물리 계층<br>(Physical Layer)|실제 장비,  회선과 관련된 특성 규정(물리적 연결)<br>물리적 데이터 전송 경로를 제공하며, 전기 신호, 빛 등으로 데이터를 전송| RS-232C, V.35, Ethernet, FDDI

# TCP/IP(Transmission Control Protocol/Internet Protocol)
TCP/IP는 인터넷에 연겨뢴 서로 다른 기종의 컴퓨터들이 데이터를 주고받을 수 있도록 하는 표준 프로토콜이다. TCP/IP 패킷 통신 방식은 인터넷 프로토콜인 IP와 전송 조절 프로토콜인 TCP로 이루어져있다. IP는 패킷 전달 여부를 보증하지 않고, 패킷을 보낸 순서와 받는 순서가 다를 수 있다. TCP는 IP 위에서 동작하는 프로토콜로, 데이터의 전달을 보증하고 보낸 순서대로 받게 해준다.

**작동 방식**<br>
TCP/IP를 사용하여 인터넷을 통해 데이터를 전송하면 데이터가 더 작은 패킷으로 나누어진다. 각 패킷에는 대상 주소, 소스 주소 및 실제 데이터가 포함된다. 그런 다음 이러한 패킷은 네트워크를 통해 전송되고 수신 측에서 재조립된다. TCP는 패킷이 올바른 순서로 도착하는지 확인하고 손실되거나 손상된 경우 재전송을 요청한다.

**장점**
1. 광범위한 채택과 호환성
    - 사실상 모든 장치와 운영 체제에서 지원된다.
2. 오류 검사 및 혼잡 제어 메커니즘을 제공
    - 안정적이고 효율적인 데이터 전송을 보장한다.
3. 모듈식 설계
    - 쉽게 확장하고 새로운 프로토콜


# Subnet
서브넷, 즉 서브네트워크는 네트워크 내부의 네트워크이다. 서브넷은 네트워크를 보다 효율적으로 만든다. 서브넷을 통해 네트워크 트래픽은 불필요한 라우터를 통과하지 않고 더 짧은 거리를 이동하여 대상에 도달할 수 있다.

예를 들어, 채율이가 바로 옆 마을에 사는 단희에게 보내는 편지를 우체통에 넣었을 때 편지가 최대한 빨리 단희에게 도착하려면 채율이의 우체국에서 단희의 마을에 있는 우체국으로 배달된 다음 단희에게 배달되어야 한다. 편지가 수백 미터 떨어진 우체국으로 처음 보내지면 채율이의 편지가 단희에게 도착하는 데 훨씬 더 오래 걸릴 수 있다.

우편 서비스와 마찬가지로 네트워크는 메시지가 가능한 한 직접 이동할 때 더 효율적이다. 네트워크에서 다른 네트워크로부터 데이터 패캣을 수신하면 패킷이 대상까지 비효율적인 경로를 사용하지 않도록 서브넷별로 해당 패킷을 정렬하고 라우팅한다.

- 서브네팅: 네트워크를 분할하는 것을 의미한다. 이것은 서브넷 마스크(Subnet Mask)를 통해서 가능하다.

### Subnet Mask
서브넷 마스크는 IP 주소와 비슷하지만, 네트워크 내에서 내부적으로만 사용된다. 라우터는 서브넷 마스크를 사용하여 데이터 패킷을 올바른 위치로 라우팅한다. 서브넷 마스크는 인터넷을 통과하는 데이터 패킷 내에서 표시되지 않으며, 이러한 패킷은 라우터가 서브넷과 일치하는 대상 IP 주소만 나타낸다.

# DHCP(Dynamic Host Configuration Protocol)
네트워크 장비들에게 동적으로 IP 주소와 기타 네트워크 설정 정보를 자동으로 할당하는 프로토콜이다. DHCP 서버는 클라이언트에게 IP 주소를 임대 방식으로 제공하며, 서브넷 마스크, 게이트웨이, DNS 서버 정보도 함께 제공한다. DHCP는 네트워크 관리자가 해야 할 작업을 간소화한다. 

- 기본 게이트웨이: 로컬 네트워크와 인터넷 간에 데이터를 주고 받는 기본 게이트웨이
- DNS 서버: IP 주소의 이름을 사람이 기억할 수 있는 이름으로 변환하는 DNS 서버

#### 장점
1. 신뢰성 높은 DHCP IP 주소 구성
    - DHCP는 동일한 IP 주소를 이용하는 두 명의 사용자 사이의 충돌을 방지하도록 도와준다.
2. 높은 이동성
    - DHCP는 높은 이동성을 보장하며 사용자는 네트워크 범위 내에서 어디서든지 모바일 장치를 이용할 수 있다.
3. 효율적인 네트워크 관리
    - DHCP를 사용하면 별도의 IP 할당 서버가 필요하지 않아 네트워크 관리 효율성이 개선된다.
4. IP 체계의 유연성
    - DHCP를 이용하면 최종 사용자에게 지장을 주지 않으면서 IP 주소 체계를 손쉽게 변경할 수 있다.

#### 단점
1. 보안 문제
    - DHCP는 아무런 인증 과정을 거치지 않기 때문에 사이버 공격에 취약하다. 

### 작동 방식
1. DHCP Discover
    - Host -> Server
    - Host가 DHCP 서버를 찾기 위해 LAN 상에 브로드캐스팅하여 DHCP 서버를 찾는 메세지를 보낸다.
2. DHCP Offer
    - Host <- Server
    - 메세지 내의 Broadcast Flag 값에 따라 달라지는데, 이 Flag=1이면 DHCP Offer 메세지를 Broadcast로, Flag=0이면 Unicast로 보낸다.
    - DHCP 서버가 호스트에게 자신의 존재를 알리고, 호스트에게 할당할 IP 주소 정보를 포함한 다양한 네트워크 정보를 같이 호스트에게 전달한다.
3. DHCP Request
    - Host -> Server
    - 호스트는 DHCP 서버들의 존재를 알았고, DHCP 서버가 호스트에게 제공할 네트워크 정보를 알았으니 호스트는 DHCP Request 메시지를 통해 하나의 DHCP 서버를 선택하고 해당 서버에게 호스트가 사용할 네트워크 정보를 요청한다.
4. DHCP ACK
    - Host <- Server
    - DHCP 절차의 마지막 메시지로 DHCP 서버가 호스트에게 네트워크 정보를 전달해주는 메시지이다. DHCP Offer의 네트워크 정보와 동일한 파라미터가 포함된다. 
    - DHCP Offer와 같이 Unicast이거나 Broadcast일 수 있다.

# CIDR 표기법
DICR 표기법은 IP 주소 뒤에 슬래스(/)와 숫자를 붙여 사용한다.

    192.168.1.0/24

여기에서 /24는 앞 24비트가 네트워크 주소임을 의미하며, 나머지 8비트는 호스트 주소로 사용된다는 것을 알 수 있다. 이는 서브넷 마스크 255.255.255.0과 동일한 의미를 가지며, 네트워크 내에서 사용할 수 있는 IP 개수를 쉽게 계산할 수 있다.

/24 라는 뜻은 32비트 중 앞에서부터 차례대로 1의 개수가 24개라는 뜻이다.
/24 -> 11111111.111111111.11111111.00000000

클래스 | CIDR 표기

클래스 A | 0.0.0.0/8 ~ 127.255.255.255/8
클래스 B | 128.0.0.0/16 ~ 191.255.255.255/16
클래스 C | 192.0.0.0/24 ~ 223.255.255.255/24
클래스 D | 224.0.0.0/4

# ARP/RARP
## ARP(Address Resolution Protocol)
ARP는 주소 결정 프로토콜이라고 불린다. 내가 통신해야 할 상대방의 MAC 주소를 모를 때 상대방의 IP 주소로 MAC 주소를 물어 볼 목적으로 만들어진 프로토콜이다. 즉, 논리적 주소인 IP 주소를 물리적 주소인 MAC 주소에 대응시키기 위해 사용된다. 

ARP가 필요한 이유는 IP 주소와 MAC 주소로만 통신할 수 없기 때문이다.

### ARP 흐름
ARP의 동작 방식은 크게 두 가지로, ARP Request와 ARP Replay로 나뉜다.

ARP Request는 ARP를 요청하는 것이다. 송신자는 목적지의 MAC 주소를 알아내기 위해, Broadcast 방식으로 요청을 보낸다. 반대로 ARP Replay는 ARP에 대한 응답이다. 목적지의 MAC 주소를 알아내었으면, 알아낸 MAC 주소를 송신자에게 전달하는 것이다. 이때는 Unicast 방식으로 전달한다.

- Broadcast 방식: 모든 호스트 및 라우터에게 접근하는 것이다.
- Unicast 방식: 하나의 단말에만 접근하는 것이다.

## RARP(Reverse ARP)
말 그대로 반대로 동작하는 ARP인데, RARP는 IP 주소가 정해져 있지 않은 단말이 IP 할당을 요청할 때 사용한다.
