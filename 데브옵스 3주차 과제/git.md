# GIT 기본 개념

### git 의미
    git이란 분산형 버전 관리 시스템의 한 종류이다.

## git 흐름 이해

1. git pull
    
    먼저 원격 저장소에서 최신 코드를 받아온다. 이를 통해 현재 로컬 코드가 최신 상태인지 확인하고, 다른 사람이 진행한 작업을 로컬 리포지토리에 포함시킨다.
    
2. git checkout -b add_profile_page
    
    새로운 기능을 위한 브랜치를 생성한다. 이 브랜치는 사용자 프로필 페이지를 추가하는 작업을 포함한다.
    
3. git add-A
    
    새로운 프로필 페이지에 필요한 모든 파일과 변경사항을 staging area에 추가한다.
    
4. git commit -m “feat:add user profile page”
    
    staging area에 있는 변경사항을 커밋하여 작업 내용에 대한 스냅샷을 생성한다.
    
5. git push
    
    커밋한 변경 사항을 원격 저장소에 푸시한다.

## git flow

    고수준으로 저장소를 관리하기 위한 git의 확장 개념이다.

- feature
    
    특정 기능을 개발/수정할 때 사용된다.
    
- release
    
    배포시에 사용된다.
    
- hotfix
    
    서비스 되고 있는 소스의 문제를 해결 후 병합해야할 때 사용된다.

## github 의미

- 의미
    
    git을 사용하는 프로젝트를 지원하는 웹호스팅 서비스이다.

<br/>

## git vs github

    git과 github는 완전히 다른 도구이다. git은 로컬 환경에서 버전을 관리하는 시스템인 반면, github는 git을 기반으로 협업과 코드 공유를 위한 클라우드 플랫폼이다.
