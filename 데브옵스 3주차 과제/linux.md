# inux

## 1. /etc 디렉토리 핵심 파일 조사

### DNS 동작 흐름 설명
1. 브라우저가 로컬 DNS 캐시를 먼저 확인한다.
2. 없으면, ISP의 리커시브 DNS 서버에 요청
    - `리커시브 DNS 서버: 사용자를 대신해 모든 DNS 요청을 처리`

3. 리커시브 서버는 루트 네임서버로 쿼리 전달

    - `루트 네임서버: 모든 TLD 서버의 위치를 알려줌`


4. 루트 서버 -> TLD 서버(.com, .org 등)로 안내

    - `TLD 네임서버: 해당 도메인의 네임서버 정보를 제공`

5. TLD 서버 -> 해당 도메인의 권한 네임서버 위치 제공
    - `권한 네임서버: 최종적으로 실제 IP 주소를 보유한 서버`
6. 권한 네임서버가 실제 IP 주소 응답
7. 브라우저가 응답받은 IP로 웹사이트 접송


### hosts vs DNS 차이

**hosts 파일** <br>
도메인 이름과 그에 해당하는 IP주소를 직접 매핑해 둔 텍스트 파일이다.

- 위치
    - /ect/hosts
- 역할
    - 특정 이름을 IP 주소로 변환해야 할 때, DNS 서버에 묻기 전에 가장 먼저 확인하는 곳이다.

**우선순위**
1. /ect/hosts 파일
    - 가장 먼저 이 로컬 주소록을 확인
2. DNS 서버
    - /ect/hosts에 찾는 이름이 없다면, 그때서야 /etc/resolv.conf에 등록된 외부 DNS 서버에게 질문을 던진다.


## 2. 네트워크 설정 파일 분석

### UID / GID 개념
**UID**<br>
계정을 식별하기 위한 고유 번호를 나타내며 0~32767의 숫자로 나타낸다.
  - 0 (root)은 리눅스 시스템의 최고관리자 계정을 뜻한다.

**GID**
사용자 계정이 속한 그룹의 고유 번호. 한 그룹에 여러 계정이 담겨있을 수 있어 /etc/passwd 파일 내에는 번호가 중복되어 보일 수 있다. 하지만 각 그룹끼리는 고유 번호로 나뉜다.


### root 계정 특징

**root** <br>
시스템을 관리할 수 있는 관리자 권한의 계정이자 슈퍼 유저
리눅스 파일 체제의 최상위 디렉토리(/)로도 표현한다.

root 권한이 있으면 모든 파일과 디렉토리에 대해 읽고 쓸 수 있고, 생성할 수도 있지만 제거할 수도 있다. 시스템 구성을 변경할 수도 있다. 그래서 매우 편하지만 조심히 행동해야 하는 계정이다.

- sudo
  - 현재 계정에서 일시적으로 root 권한이 필요할 경우에 사용하는 명령어

- su root / su -root
   - 현재 계정이 root 계정으로 전환된다.
