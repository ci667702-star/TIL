# SSH란?
**SSH(Secure Shell)의 의미**<br>
원격지 호스트 컴퓨터에 접속하기 위해 사용되는 인터넷 프로토콜이다. SSH는 Telnet에 보안 기능이 추가된 암호화 버전이다. 또한 SSH는 포트 포워딩을 허용한다. SSH는 서버를 원격으로 제어하고, 인프라를 관리하며, 파일을 전송하는 데 자주 사용된다. 

예를 들어 여행 중일 때 소유자는 자신이 없는 동안 매장을 원활하게 운영하기 위해 직원에게 먼 곳에서 지시할 수 있다. 

**SSH와 Telnet의 차이점**
구분 | SSH | Telnet
:--: | :--|:--
보안 | 모든 데이터를 암호화하여 전송 | 평문으로 데이터를 전송
사용 용도 | 보안이 중요한 환경에서 사용 | 주로 로컬 네트워크나 보안 요구가 낮은 환경에서 사용
포트 번호 | 22 | 23
성능 | 상대적으로 느림 | 빠름

## SSH의 특징
1. 강력한 암호화
    - SSH는 모든 클라이언트와 서버 간의 통신을 암호화하여 보안을 강화한다. 이로 인해 네트워크 상에서 데이터가 탈취되더라도, 암호화되어 있어 원래의 데이터를 복구하는 것이 매우 어렵다.

암호화 기법 | 설명
:--:|:--
대칭키 암호화 | 같은 키를 사용하여 데이터를 암호화하고 복호화 함
비대칭키 암호화 | 공개키와 개인키를 사용하여 안전한 인증을 수행함
해시 알고리즘 | 데이터 무결성을 보장하기 위해 해시값을 사용함

2. 강력한 사용자 인증
    - SSH는 사용자와 서버를 인증하여 보안을 강화한다. 초기 접속 시 일반적으로 비밀번호를 통해 인증하며, 추가적으로 공개키 기반의 인증법도 지원한다. 이는 계정 도용을 미연에 방지하는 데에 매우 유용하다.
3. 다양한 플랫폼 지원
    - SSH는 Unix 계열 시스템, Windows, macOS 등 다양한 플랫폼에서 사용할 수 있다. 이로 인해 사용자는 자신이 사용하는 운영체제에 상관없이 안전한 원격 접속과 데이터 전송을 수행할 수 있다.
4. Default로 22번 포트를 사용한다.

## SSH 구성 요소
- SSH Client
    - SSH 프로토콜을 이용하여 원격 시스템(SSH Server)에 접속 및 명령을 내리도록 하는 주체
- SSH Server
    - SSH Client로부터 SSH 프로토콜을 통해 받은 명령을 수행하는 주체이다. 쉽게 말해, 원격지 서버 또는 컴퓨터를 말한다.

## SSH의 기능
1. 서버 관리
    - SSH는 서버의 원격 관리에 중요한 역할을 한다. 시스템 관리자들은 안전하게 원격 서버에 접속하여 서버의 설정, 로그 확인, 서비스 관리 등을 수행할 수 있다.
2. 원격 작업
    - SSH를 이용하면 지리적으로 떨어진 서버에서도 로컬에서 작업하는 것처럼 터미널을 통해 명령어를 실행할 수 있다.
3. 파일 전송
    - SSH는 SCP(보안 복사, Secure Copy)와 SFTP(보안 파일 전송 프로토콜)를 제공하여 파일을 안전하게 전송할 수 있는 기능을 제공한다.
4. 원격 암호화된 연결
    - SSH는 사용자의 장치와 멀리 떨어진 컴퓨터(대체로 서버)의 연결을 설정한다. SSH는 암호화를 사용하여 연결을 통과하는 데이터를 변환시킨다. 가로채는 쪽에서는 정적 데이터만 발견할 수 있다. 이는 해독하지 않으면 아무 의미가 없는 무작위 데이터이다. SSH는 외부인이 암호를 해독하기 매우 어려운 암호화 방법을 사용한다.
5. 터널링 기능
    - 네트워킹에서 터널링은 일반적으로 사용할 수 없는 프로토콜이나 경로를 사용하여 네트워크에서 패킷을 이동시키는 방법이다. 터널링은 데이터 패킷을 헤더라는 추가 정보로 래핑하여 목적지를 변경하는 방식으로 작동한다. SSH 터널은 포트 포워딩이라는 기술을 사용하여 한 컴퓨터에서 다른 컴퓨터로 패킷을 전송한다.
    - 포트포워딩은 두 사람 사이에 메시지를 전달하는 것과 같다. A가 B에게 메시지를 보내면 다시 B가 C에게 메시지를 전달할 수 있는데, 마찬가지로 포트포워딩은 한 컴퓨터의 IP 주소와 포트로 향하는 데이터 패킷을 다른 컴퓨터의 IP 주소와 포트로 전송한다.

종류 | 옵션 | 설명 
:--|:--|:--
Local Forwarding | -L | 로컬에서 지정한 포트를 통해 원격 서버의 특정 포트에 접근할 수 있도록 터널을 만든다.
Remote Forwarding | -R | 원격 서버에서 로컬 서버의 특정 포트에 접근하도록 한다.
Dynamic Forwarding | -D | SOCKS 프록시를 만들어 로컬의 트래픽을 만들어 암호화된 SSH 연결을 통해 라우팅한다.

>ssh -L ([로컬 ip]:)[로컬 포트]:[원격지 ip]:[원격지 포트] [SSH 서버 username]@[SSH 서버 ip]

이 로컬 터널링을 통해 로컬 포트로 전달되는 트래픽은 [원격지 ip]:[원격지 포트]로 포워딩 된다. 로컬 ip 구문은 생략 가능하다.

- 패킷:인터넷 내에서 데이터를 보내기 위한 경로배정을 효율적으로 하기 위해서 데이터를 여러 개의 조각들로 나누어 전송하는데, 이때 이 조각을 패킷이라고 한다.

### SCP와 SFTP
**SFTP(Secure File Transfer Protocol)**<br>
SSH를 기반으로 파일을 안전하게 전송하는 프로토콜이다. FTP의 기능을 SSH로 감싸서 보안을 강화한 방식이다.

*장점*
1. SSH를 통해 모든 데이터와 명령어가 암호화되어 전송되므로 높은 수준의 보안을 제공한다
2. SFTP는 FTP와 유사한 명령어 세트를 사용하며, 파일 및 디렉토리 조작에 다양한 명령어를 제공한다.
3. SFTP는 SSH와 동일한 포트를 사용하믈 방화벽 및 네트워크 설정이 단순하다.

*단점*
1. SSH 키의 유효성 검사 및 관리가 복잡하다.
2. 복잡한 명령어 처리 및 파일 조직 기능으로 인해 SCP보다 속도가 느릴 수 있다.

*작동 방식*
- 세션 기반
    - SFTP는 SSH 연결을 통해 세션을 설정하고, 이를 통해 파일 전송, 디렉토리 탐색, 파일 삭제 및 이름 변경 등의 작업을 수행한다.
- 명령어 세트 
    - SFTP는 FTP와 유사한 명령어 세트를 사용하며, 파일 및 디렉토리 조작에 다양한 명령어를 제공한다.
- 단일 포트 사용
    - SFTP는 SSH와 동일한 포트를 사용하므로 방화벽 및 네트워크 설정이 단순하다.

**SCP(Secure Copy Protocol)**<br>
원격 컴퓨터간에 파일을 안전하게 전송하기 위한 프로토콜이다. SSH 프로토콜을 기반으로 하며, SSH를 통해 암호화된 터널을 통해 데이터를 전송할 수 있다. 암호화된 터널을 통해 데이터를 전송하기 때문에 전송 시 제 3자에게 노출되지 않는다.

*장점*
1. SSH를 통해 모든 데이터가 암호화되어 있으므로 높은 보안을 제공한다.
2. 명령어가 단순하여 사용하기 쉽고 빠르게 파일을 전송할 수 있다.
3. 추가 기능이 없어 SFTP보다 빠른 파일 전송이 가능하다.

*단점*
1. SCP는 파일 전송에만 집중되어 있어 파일 목록 조회, 삭제, 속성 변경 등의 기능을 지원하지 않는다.
2. 전송 중 오류 발생 시 재시도 등의 기능이 부족하다.

*작동 방식*
- 단일 명령어
    - SCP는 파일 전송을 위한 단일 명령어 기반 프로토콜이다. 사용자는 단일 명령어를 통해 파일을 복사할 수 있다.
- 단방향 전송
    - SCP는 파일을 전송하거나 복사할 때 단방향 전송을 수행하며, 파일 전송 후 추가 작업은 지원하지 않는다.
- 단일 포트 사용
    - SCP 역시 SSH와 동일한 포트를 사용한다.

구분 | SCP | SFTP
:--:|:--|:--
기능 | 파일 전송만 함 | 원격 파일 관리를 위한 광범위한 작업을 제공
속도 | SFTP 보다 빠름 | 패킷의 암호화 및 일치를 기다려야 하므로 SCP보다 느림
파일 전송 용량 | 4GB 이상의 파일 전송 불가 | 4GB 이상의 파일도 전송 가능
세션 유지 | 파일 전송 취소시 세션을 취소해야함 | 파일 전송 취소시 세션을 유지할 수 있음
전송 재개 가능 | 전송 재개 불가 | 전송 재개 가능

### SSH 통신 과정
1. 클라이언트는 서버에 원격 접속하기 위해 연결을 설정하는 프로세스를 시작한다.
    - 클라이언트가 서버에 원격 접속하기 위해 서버의 TCP 22번 포트로 SSH 접속 요청을 보내는 것이 SSH 연결의 첫 단계이다. 서버는 클라이언트에게 서버가 지원하는 프로토콜의 버전을 응답으로 보내준다.
2. 서버는 자신의 공개키를 클라이언트에게 전송한다.
    - 서버는 클라이언트로부터 SSH 접속 요청을 받고 자신의 공개키를 클라이언트에게 전송하고 클라이언트는 서버로부터 받은 공개키를 로컬에 저장한다.
3. 클라이언트와 서버는 여러 parameter들을 주고 받으며 보안 채널을 확립한다.
    1. 올바른 서버인지 확인
        - 클라이언트는 SSH로 원격 접속하려고 하는 서버가 올바른 서버인지 확인할 필요가 있다. 이를 위해 클라이언트는 known_host 파일에 존재하는 서버의 공개키를 통해 정상적인 서버인지 확인하는 작업을 수행한다.
    2. 암호화된 통신을 위한 세션키 생성
        - 세션키는 대칭키로 전체 세션을 암호화하는 데에 사용되며 모든 통신을 암호화한다.
        - 이때 디피-헬만(Diffie-Hellman, DH) 알고리즘은 상대방의 공개키와 나의 개인키를 통해 대칭키를 얻는 방법을 사용한다. 
        - 이 단계에서 클라이언트와 서버는 임시 비대칭키 방식의 키 쌍을 생성하고 공개 키를 교환한다.
    3. 서버에 접근할 수 있는 클라이언트인지 확인
        - 서버 또한 자신에게 접속하려는 클라이언트가 자신에게 접근할 수 있는 권한이 있는지 확인하는 단계가 필요하다. 가장 간단한 방법으로 패스워드 인증이 있다. 하지만 보안이 약해 가장 많이 사용되는 방법은 SSH 키 쌍을 사용하는 것이다.
4. 클라이언트가 서버에 원격 접속을 할 수 있다.

- known_hosts 파일: SSH를 사용해 원격으로 호스트에 연결할 경우, 클라이언트가 연결하려는 호스트(서버)의 공개 키와 호스트명(IP 주소/도메인 이름)을 저장하는 역할을 한다.
- 디피-헬만 알고리즘: 두 파티가 안전하지 않은 채널을 통해 비밀 키를 공유할 수 있게 해주는 암호학적 프로토콜이다. 이는 키 교환 알고리즘의 한 종류로, 주로 공개키 암호 시스템에서 사용된다.

방법 | 설명
:--:|:--
Password 인증 | 사용자가 입력한 비밀번호를 Session Authentication 과정에서 생성한 Session Key로 암호화하여 SSH Server에게 보내고, 이를 SSH Server가 검증하면 끝나는 방식이다.
Key-Pair | 공개키를 서버의 authorized_keys에 미리 등록해두고 접속 시 개인키로 서명하여 인증하는 방식이다.

### SSH 알고리즘 종류
**ECDSA**<br>
타원 곡선의 수학을 사용하여 디지털 서명을 만드는 암호화 방법이다. 더 작은 키 크기로 효율성과 강력한 보안으로 유명하다. 이는 모바일 기기 및 사물 인터넷과 같이 컴퓨팅 파워와 스토리지가 제한된 환경에 특히 적합하다.

*특징*
1. 고속 연산
    - RSA에 비해 작은 키 크기로 동일한 보안성을 제공하며 연산 속도가 빠르다.
2. 서명 생성
    - 개인 키를 사용하여 특정 데이터에 대한 서명을 생성한다.
3. 서명 검증
    - 공개 키를 사용하여 서명의 유효성을 확인할 수 있다.
4. 무결성 보장
    - 데이터가 변조되지 않았음을 증명할 수 있다.

*작동 원리*
1. 키 생성
    - ECDSA는 타원 곡선 방정식을 기반으로 공개-사적 키 쌍을 생성한다.
2. 서명
    - 개인 키는 메시지를 서명하는 데 사용되어 고유한 디지털 서명을 생성한다.
3. 검증
    - 공개키는 디지털 서명의 진위를 검증하는 데 사용된다.

- 장점
    - 다양한 타원 곡선 중에서 선택할 수 있어 특정 환경에 맞게 최적화할 수 있다.
    - 타원 곡선 수학에 기반한 ECDSA는 안전하고 알려진 암호 공격에 매우 강하다.
- 단점
    - ECDSA를 올바르게 구현하는 것은 수학적으로 복잡하며 신중하게 다루지 않으면 취약점 위험이 오히려 증가할 수 있다.

*실제 사용 사례*<br>
ECDSA는 비트코인을 포함한 블록체인 기술에서 광범위하게 사용되어 거래를 서명하고 그 진위를 보장한다.


**RSA**<br>
전 세계에서 가장 많이 사용되는 공개키 암호 알고리즘이다. 안전한 데이터 전송 및 정보 보호에 널리 사용되는 암호화 기술이다. 주로 적은 양의 데이터나 전자서명에 사용한다. RSA 암호화에 사용되는 모듈러 연산은 큰 정수를 소인수분해하기가 어렵다는 특성을 이용한다. 

예를 들어, 3223이라는 숫자가 어떤 두 소수의 곱인지를 나타내려면 3223을 가장 작은 소수부터 하나씩 나눠보면서 나누어 떨어지는 숫자를 찾아야한다. 숫자가 작은 경우라면 금방 계산할 수 있지만, RSA 알고리즘에서 수백 자리의 소수를 사용하므로 많은 계산이 필요하다. 키의 길이는 1024비트부터 4096비트까지 될 수 있으며, 키의 길이가 길수록 보안성이 강화된다.

- 장점
    - 암호화 및 서명에 모두 사용할 수 있다.
    - 잘 알려진 알고리즘이다.
    - 다양한 키 길이를 사용할 수 있다.
- 단점
    - 암호화 및 복호화 과정에서 비교적 많은 시간과 계산 자원이 필요하다.
    - 양자컴퓨터 사용 시 단시간에 풀릴 것을 예상할 수 있다.

*작동 원리*
1. 키 생성
    - RSA는 암호화용 공개키와 복호화용 개인 키를 생성한다.
2. 암호화
    - 발신자는 수신자의 공개키를 사용해 메시지를 암호화한다.
3. 복호화
    - 수신자는 자신의 개인키를 사용해 메시지를 복호화한다.

*실제 사용 사례*<br>
RSA는 SSL/TLS 인증서에서 안전한 HTTPS 연결을 구축하기 위해 일반적으로 사용된다. 그리고 PGP와 같은 많은 구형 디지털 서명 시스템은 RSA를 통해 인증을 수행한다.

**Ed25519**<br>
SHA-512 및 Curve25519를 사용한 EdDSA 서명 체계이다. 또한 Ed25519는 공개키 서명 시스템이다. Ed25519 시그니처는 타원 곡선 시그니처로 여러 수준의 설계 및 구현에서 신중하게 엔지니어링되어 보안을 손상시키지 않으면서 매우 빠른 속도를 달성한다.

- 장점 
    - 매우 빠르고 효율적이다
    - 모바일 기기와 같은 저성능 환경에서 유용하다
- 단점
    - 서명에만 사용할 수 있다.
    - 오래된 시스템에서는 지원하지 않을 수 있다.

*기술적 특징*
1. 타원 곡선 
    - 소인수분해 대신 타원 곡선상의 이산 로그 문제를 푼다.
2. 결정론적 서명
    - 서명할 때 난수 생성기에 의존하지 않는다.
3. 상수 시간
    - 연산 시간이 입력값에 따라 달라지지 않는다.

*실제 사용 사례*<br>
보안 통신 전반(암호화 메시징, 이메일 보안, 안전한 파일 전송)에서 Ed25519는 빠른 서명/검증과 강한 보안성으로 널리 사용된다. 결정적 특성은 구현을 단순화하고 난수 관련 취약점 리스크를 낮춘다.

블록 체인/가상자산에서도 거래 서명/검증 효율이 네트워크 속도와 보안 유지에 핵심이다. 소형 키와 빠른 성능은 자원이 제한된 IoT/임베디드 환경에서 특히 유리하다.
