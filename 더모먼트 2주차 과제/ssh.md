# SSH란?
**SSH(Secure Shell)의 의미**<br>
원격지 호스트 컴퓨터에 접속하기 위해 사용되는 인터넷 프로토콜이다. SSH는 Telnet에 보안 기능이 추가된 암호화 버전이다. 또한 SSH는 포트 포워딩을 허용한다. SSH는 서버를 원격으로 제어하고, 인프라를 관리하며, 파일을 전송하는 데 자주 사용된다. 

예를 들어 여행 중일 때 소유자는 자신이 없는 동안 매장을 원활하게 운영하기 위해 직원에게 먼 곳에서 지시할 수 있다. 

## SSH의 특징
1. 강력한 암호화
    - SSH는 모든 클라이언트와 서버 간의 통신을 암호화하여 보안을 강화한다. 이로 인해 네트워크 상에서 데이터가 탈취되더라도, 암호화되어 있어 원래의 데이터를 복구하는 것이 매우 어렵다.

암호화 기법 | 설명
:--:|:--
대칭키 암호화 | 같은 키를 사용하여 데이터를 암호화하고 복호화 함
비대칭키 암호화 | 공개키와 개인키를 사용하여 안전한 인증을 수행함
해시 알고리즘 | 데이터 무결성을 보장하기 위해 해시값을 사용함

2. 강력한 사용자 인증
    - SSH는 사용자와 서버를 인증하여 보안을 강화한다. 초기 접속 시 일반적으로 비밀번호를 통해 인증하며, 추가적으로 공개키 기반의 인증법도 지원한다. 이는 계정 도용을 미연에 방지하는 데에 매우 유용하다.
3. 다양한 플랫폼 지원
    - SSH는 Unix 계열 시스템, Windows, macOS 등 다양한 플랫폼에서 사용할 수 있다. 이로 인해 사용자는 자신이 사용하는 운영체제에 상관없이 안전한 원격 접속과 데이터 전송을 수행할 수 있다.

## SSH 구성 요소
- SSH Client
    - SSH 프로토콜을 이용하여 원격 시스템(SSH Server)에 접속 및 명령을 내리도록 하는 주체
- SSH Server
    - SSH Client로부터 SSH 프로토콜을 통해 받은 명령을 수행하는 주체이다. 쉽게 말해, 원격지 서버 또는 컴퓨터를 말한다.

## SSH의 기능
1. 서버 관리
    - SSH는 서버의 원격 관리에 중요한 역할을 한다. 시스템 관리자들은 안전하게 원격 서버에 접속하여 서버의 설정, 로그 확인, 서비스 관리 등을 수행할 수 있다.
2. 원격 작업
    - SSH를 이용하면 지리적으로 떨어진 서버에서도 로컬에서 작업하는 것처럼 터미널을 통해 명령어를 실행할 수 있다.
3. 파일 전송
    - SSH는 SCP(보안 복사, Secure Copy)와 SFTP(보안 파일 전송 프로토콜)를 제공하여 파일을 안전하게 전송할 수 있는 기능을 제공한다.
4. 원격 암호화된 연결
    - 특징 1번 참조.
5. 터널링 기능
    - 네트워킹에서 터널링은 일반적으로 사용할 수 없는 프로토콜이나 경로를 사용하여 네트워크에서 패킷을 이동시키는 방법이다. 터널링은 데이터 패킷을 헤더라는 추가 정보로 래핑하여 목적지를 변경하는 방식으로 작동한다. SSH 터널은 포트 포워딩이라는 기술을 사용하여 한 컴퓨터에서 다른 컴퓨터로 패킷을 전송한다.
    - 포트포워딩은 두 사람 사이에 메시지를 전달하는 것과 같다. A가 B에게 메시지를 보내면 다시 B가 C에게 메시지를 전달할 수 있는데, 마찬가지로 포트포워딩은 한 컴퓨터의 IP 주소와 포트로 향하는 데이터 패킷을 다른 컴퓨터의 IP 주소와 포트로 전송한다.

종류 | 옵션 | 설명 
:--|:--|:--
Local Forwarding | -L | 로컬에서 지정한 포트를 통해 원격 서버의 특정 포트에 접근할 수 있도록 터널을 만든다.
Remote Forwarding | -R | 원격 서버에서 로컬 서버의 특정 포트에 접근하도록 한다.
Dynamic Forwarding | -D | SOCKS 프록시를 만들어 로컬의 트래픽을 만들어 로컬의 트래픽을 암호화된 SSH 연결을 통해 라우팅한다.

>ssh -L ([로컬 ip]:)[로컬 포트]:[원격지 ip]:[원격지 포트] [SSH 서버 username]@[SSH 서버 ip]

이 로컬 터널링을 통해 로컬 포트로 전달되는 트래픽은 [원격지 ip]:[원격지 포트]로 포워딩 된다. 로컬 ip 구문은 생략 가능하다.

- `패킷:인터넷 내에서 데이터를 보내기 위한 경로배정을 효율적으로 하기 위해서 데이터를 여러 개의 조각들로 나누어 전송하는데, 이때 이 조각을 패킷이라고 한다.` 

### SSH 통신 과정
1. 클라이언트는 서버에 원격 접속하기 위해 연결을 설정하는 프로세스를 시작한다.
    - 클라이언트가 서버에 원격 접속하기 위해 서버의 TCP 22번 포트로 SSH 접속 요청을 보내는 것이 SSH 연결의 첫 단계이다. 서버는 클라이언트에게 서버가 지원하는 프로토콜의 버전을 응답으로 보내준다.
2. 서버는 자신의 공개키를 클라이언트에게 전송한다.
    - 서버는 클라이언트로부터 SSH 접속 요청을 받고 자신의 공개키를 클라이언트에게 전송하고 클라이언트는 서버로부터 받은 공개키를 로컬에 저장한다.
3. 클라이언트와 서버는 여러 parameter들을 주고 받으며 보안 채널을 확립한다.
    1. 올바른 서버인지 확인
        - 클라이언트는 SSH로 원격 접속하려고 하는 서버가 올바른 서버인지 확인할 필요가 있다. 이를 위해 클라이언트는 known_host 파일에 존재하는 서버의 공개키를 통해 정상적인 서버인지 확인하는 작업을 수행한다.
    2. 암호화된 통신을 위한 세션키 생성
        - 세션키는 대칭키로 전체 세션을 암호화하는 데에 사용되며 모든 통신을 암호화한다.
        - 이때 디피-헬만(Diffie-Hellman, DH) 알고리즘은 상대방의 공개키와 나의 개인키를 통해 대칭키를 얻는 방법을 사용한다. 
        - 이 단계에서 클라이언트와 서버는 임시 비대칭키 방식의 키 쌍을 생성하고 공개 키를 교환한다.
    3. 서버에 접근할 수 있는 클라이언트인지 확인
        - 서버 또한 자신에게 접속하려는 클라이언트가 자신에게 접근할 수 있는 권한이 있는지 확인하는 단계가 필요하다. 가장 간단한 방법으로 패스워드 인증이 있다. 하지만 보안이 약해 가장 많이 사용되는 방법은 SSH 키 쌍을 사용하는 것이다.
4. 클라이언트가 서버에 원격 접속을 할 수 있다.

방법 | 설명
:--:|:--
Password 인증 | 사용자가 입력한 비밀번호를 Session Authentication 과정에서 생성한 Session Key로 암호화하여 SSH Server에게 보내고, 이를 SSH Server가 검증하면 끝나는 방식이다.
Key-Pair | SSH Client에서 Public Key, Private Key를 생성하고 이를 SSH Server에 접속할 때 Public Key를 제출하여 인증되는 방식이다.
